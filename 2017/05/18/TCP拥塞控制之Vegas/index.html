TCP拥塞算法之：Vegas

发表于 2017-05-18 | 分类于 TCP/IP |

  vegas是一种主动调整cwnd的拥塞控制算法，主要思想是设置两个阈值,alpha和beta,然后通过计算目标速率(Expected)和实际速率(Actual)差(diff)，再比较diff与alpha和beta的关系，对cwnd进行调节。vegas这篇paper的主要贡献是提供了一种拥塞检测的方法。

区分拥塞还是非拥塞状态

定义理论速率Expected和实际速率Actual：

$$Expected = cwnd/BaseRTT$$
$$Actual = cwnd/RTT$$
这里cwnd是当前的TCP窗口大小，BaseRTT是测量的最小往返时间，RTT是所测量的经平滑后的往返时间。

定义速率差Diff

$$Diff = Expected - Actual$$
$$diff = Diff*baseRTT$$

设置alpha和beta的值：

alpha = 2;
beta = 4;

基于链路状态的拥塞控制

慢开始算法(cwnd <= ssthresh)

  慢开始算法同Reno算法的慢开始的处理方法一样，每收到一个ACK，cwnd加1，这样cwnd会程指数增长，即每经过一个RTT，
$$cwnd = cwnd*2$$

自适应增长算法(cwnd > ssthresh)

Reno的处理方法:
对于cwnd而言，会设置一个ssthresh(slow-start threshold)来控制cwnd的增长。
当cwnd >ssthresh，会降低cwnd的增长速度，避免出现拥塞(congestion)具体的控制方法是，每经过一个RTT，
$$cwnd = cwnd + 1$$
也可以这样理解，每收到一个ACK，

总结
